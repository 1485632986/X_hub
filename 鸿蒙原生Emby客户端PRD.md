## 1. 项目背景与目标

### 1.1 为什么要在鸿蒙端做这样一款应用

**市场背景**

随着智能家居和私有云存储的普及，越来越多的用户选择自建家庭媒体服务器来管理电影、电视剧、音乐等多媒体内容。Emby、Jellyfin 作为开源且功能强大的媒体服务器解决方案，在全球范围内拥有数百万用户群体。在国内市场，Jellyfin 因其完全免费开源的特性，用户增长速度尤为显著。

然而，在鸿蒙生态中，用户面临一个明显的痛点：**缺乏一款专业级原生客户端**。目前用户只能通过以下方式勉强使用：

- 使用 Web 浏览器访问，体验不佳
- 借助第三方应用，但适配差、稳定性问题频发
- 转而使用其他平台设备

**竞争格局分析**

| 产品 | 平台 | 优势 | 不足 |
|------|------|------|------|
| Infuse | iOS/macOS/tvOS | 体验标杆，Apple生态深度整合 | 仅限苹果，Android/鸿蒙不可用 |
| VidHub | iOS | 价格优势，功能较全 | 非原生鸿蒙，流畅度存疑 |
| Jellyfin/MPV | Android | 开源免费 | UI简陋，交互体验差 |

**鸿蒙机遇**

华为鸿蒙生态正在快速发展，HarmonyOS NEXT 已正式商用，鸿蒙原生应用生态正处于快速扩张期。此时进入市场，可以：

1. **抢占先发优势**：成为首批专业级鸿蒙媒体播放客户端
2. **契合用户需求**：解决现有用户的实际痛点
3. **拓展生态价值**：与华为手机、平板、智慧屏形成协同

### 1.2 V1.0 核心目标

**主要目标**

打造一款稳定、流畅、功能完整的鸿蒙原生媒体服务器客户端，连接 Emby/Jellyfin 服务器，实现本地化的优质观影体验。

**关键结果（KR）**

| 指标 | 目标值 |
|------|--------|
| 服务器连接成功率 | ≥ 98% |
| 首屏加载时间 | ≤ 2秒 |
| 视频播放启动时间 | ≤ 3秒（局域网环境） |
| 主流设备兼容性 | 覆盖华为 Mate 60/Pura 70/ MatePad/智慧屏 |
| 崩溃率 | ≤ 0.1% |

## 2. 目标用户画像

### 2.1 主要用户群体

**用户群体 A：技术型影音爱好者**

- **设备偏好**：华为手机（Mate系列）+ 华为平板 MatePad + 华为智慧屏
- **用户特征**：35-50岁男性，具备一定技术能力，已搭建家庭NAS或媒体服务器
- **核心痛点**：
  - 现有解决方案无法在鸿蒙设备上流畅运行
  - 需要频繁在手机、平板、TV之间切换观看
  - 对字幕、音轨选择有较高要求
- **付费意愿**：中高，愿意为优质体验付费
- **使用场景**：下班后在家观影，周末全家娱乐

**用户群体 B：家庭多媒体管理者**

- **设备偏好**：华为智慧屏 + 华为平板
- **用户特征**：30-45岁，负责家庭影视内容管理
- **核心痛点**：
  - 希望家人能便捷访问媒体库
  - 需要管理多用户播放进度
  - 孩子观影需要家长控制功能
- **付费意愿**：中等，功能实用即可
- **使用场景**：客厅娱乐，亲子观影

**用户群体 C：轻度尝鲜用户**

- **设备偏好**：华为手机
- **用户特征**：25-35岁，刚接触 Jellyfin/Emby
- **核心痛点**：
  - 不知如何选择合适的客户端
  - 对技术细节了解有限，需要简单易用的产品
- **付费意愿**：较低，免费优先
- **使用场景**：碎片时间观影

### 2.2 目标设备清单

## 5. 非功能性需求

### 5.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| App 冷启动时间 | ≤ 2秒 | 从点击图标到显示首页 |
| 媒体库首页加载 | ≤ 3秒 | 含海报墙渲染 |
| 视频播放启动 | ≤ 3秒（局域网）/ ≤ 5秒（远程） | 从点击播放到首帧显示 |
| 切换页面响应 | ≤ 100ms | 页面切换无感知延迟 |
| 内存占用 | ≤ 200MB（闲置）/ ≤ 500MB（播放中） | 避免内存溢出 |
| 帧率稳定性 | ≥ 30fps（UI）/ 保持视频原始帧率 | 确保流畅播放 |

### 5.2 安全需求

| 需求项 | 详细说明 | 优先级 |
|--------|----------|--------|
| 敏感数据加密 | 账号密码使用 HarmonyOS 关键资产存储（similar to iOS Keychain） | P0 |
| 通信加密 | 所有网络请求强制 HTTPS，兼容自签名证书（可选信任） | P0 |
| App 锁 | 支持设置 App 启动密码、指纹、人脸解锁 | P1 |
| 安全退出 | 退出登录时清除本地缓存的认证信息 | P0 |
| 隐私保护 | 不收集用户媒体库内容元数据，仅收集必要的使用统计数据 | P1 |

### 5.3 兼容性要求

| 维度 | 要求 |
|------|------|
| HarmonyOS 版本 | 兼容 HarmonyOS 4.0 及以上版本，优先适配 HarmonyOS NEXT |
| 屏幕适配 | 支持手机（竖屏）、平板（横竖屏）、智慧屏（横屏）的响应式布局 |
| 设备能力 | 最低支持 4GB RAM 设备，优先利用 NPU/GPU 能力 |
| 视频格式 | 支持 H.264/H.265/VP9/AV1 硬解，支持 HDR10/HLG |
| 音频格式 | 支持 AAC/FLAC/DTS/Dolby 全系列音频解码与透传 |

### 5.4 可靠性要求

| 指标 | 要求 |
|------|------|
| 崩溃率 | ≤ 0.1%（千分之一） |
| ANR 率 | ≤ 0.05% |
| 核心功能可用性 | ≥ 99.5%（服务器正常情况下） |
| 断网恢复 | 网络恢复后自动重连，无需手动操作 |

## 7. 鸿蒙特色优势分析

### 7.1 一次开发多端部署

**机遇**

鸿蒙的核心优势之一是「一次开发，多端部署」。利用 ArkUI 的响应式设计，可以实现：

| 设备类型 | 适配策略 |
|----------|----------|
| 手机 | 单栏布局，竖屏优先，海报 2-3 列 |
| 平板 | 双栏布局，左侧导航+右侧内容，支持手势分屏 |
| 智慧屏 | 远场交互优化，大字体/大按钮，支持遥控器/语音 |

**实现方式**

```typescript
// 使用 ArkUI 的响应式断点机制
@Extend(Column) function adaptiveLayout() {
  .width('100%')
  .height('100%')
}
```

通过媒体查询和布局容器实现一套代码适配三种设备形态，大幅降低开发和维护成本。

### 7.2 分布式能力协同

**华为生态联动**

鸿蒙的分布式能力可以实现独特体验：

| 场景 | 功能描述 |
|------|----------|
| 跨设备续播 | 手机上看的电影，可以一键流转到智慧屏继续播放 |
| 无感配网 | 与智慧屏在同一华为账号下时，自动发现并可快速投屏 |
| 分布式算力 | 平板播放时，可调用手机算力进行视频解码预处理 |

### 7.3 原子化服务

**免安装体验**

未来可考虑将「快速播放最近内容」能力抽离为原子化服务，用户无需打开 App 即可从桌面卡片直接启动最近播放的视频。

### 7.4 参考 Infuse 的交互设计

| Infuse 特色 | 鸿蒙化适配方案 |
|------------|----------------|
| 精美海报墙动效 | 使用 ArkUI 动效框架实现流畅的微交互动画 |
| 底部播放控制栏 | 适配为鸿蒙风格的 Mini 播放器组件 |
| 文件夹式导航 | 使用鸿蒙 Navigation 组件实现多层级导航 |
| 精美的详情页布局 | 参考 Infuse 的沉浸式详情页设计，但遵循鸿蒙设计规范 |

### 7.5 参考 VidHub 的功能布局

| VidHub 特色 | 鸿蒙化适配方案 |
