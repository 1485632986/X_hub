# 鸿蒙原生 Emby 播放器开发输入文档

**版本**：v1.0  
**日期**：2024年  
**技术栈**：HarmonyOS NEXT / ArkUI / ArkTS / DevEco Studio  
**目标平台**：鸿蒙手机、平板

---

## 1. 核心用户故事（User Story）

### 1.1 添加服务器
**用户故事**：作为用户，我希望能够手动添加 Emby 服务器，包括服务器地址、端口、用户名和密码，以便完成登录认证并访问我的媒体库。  
**验收标准**：
- 支持输入服务器地址（IP/域名）
- 支持自定义端口（默认 8096）
- 支持用户名+密码认证
- 支持记住登录状态
- 连接失败时显示友好错误提示

### 1.2 浏览媒体库
**用户故事**：作为用户，我希望能够查看 Emby 服务器上的媒体库分类（电影、剧集、音乐等），以便快速找到想看的内容。  
**验收标准**：
- 展示媒体库分类卡片（电影、剧集、音乐、图片等）
- 支持网格/列表布局切换
- 支持按字母、时间、评分等排序
- 分类横滑浏览体验流畅

### 1.3 搜索影片
**用户故事**：作为用户，我希望能够通过关键字搜索 Emby 服务器上的媒体内容，以便快速定位目标影片。  
**验收标准**：
- 支持实时搜索（输入即搜索）
- 支持按类型筛选（电影/剧集/音乐）
- 搜索结果展示海报、标题、类型
- 空结果时显示友好提示

### 1.4 播放视频
**用户故事**：作为用户，我希望点击影片后能够流畅播放视频，支持直接流播放。  
**验收标准**：
- 点击播放按钮后 3 秒内开始播放
- 支持 MP4、MKV、TS、MOV、WEBM 格式
- 支持硬解码（H.264/H.265/VP9/AV1）
- 支持 HDR（Dolby Vision/HDR10/HDR10+/HLG）
- 播放过程中切后台不中断

### 1.5 切换音轨/字幕
**用户故事**：作为用户，我希望在播放过程中能够切换音频轨道和字幕轨道。  
**验收标准**：
- 支持多音轨切换（普通话/英语/粤语等）
- 支持字幕开关
- 支持字幕轨道切换（SRT/ASS/VTT）
- 支持字幕样式调节（大小、颜色、位置）

### 1.6 同步播放进度
**用户故事**：作为用户，我希望能够在不同设备间同步播放进度。  
**验收标准**：
- 播放进度自动同步到 Emby 服务器
- 支持从上次中断位置继续播放
- 支持进度同步开关（用户可控）
- 切换设备后播放位置误差 < 10 秒

## 3. 关键 API 接口梳理（Emby 官方 API）

### 3.1 认证相关

| 接口名称 | 方法 | 路径 | 主要参数 | 返回格式 | 鸿蒙 http 模块注意事项 |
|---------|------|------|---------|---------|----------------------|
| 服务器信息 | GET | /System/Info | 无 | JSON | 返回 ServerId 用于后续认证 |
| 登录认证 | POST | /Users/AuthenticateByName | Username, Pw (md5) | JSON {AccessToken, UserId} | 需在请求头携带 X-Emby-Authorization |
| 获取用户信息 | GET | /Users/{UserId} | UserId | JSON | 用于获取用户头像、名称等 |

### 3.2 媒体库相关

| 接口名称 | 方法 | 路径 | 主要参数 | 返回格式 | 鸿蒙 http 模块注意事项 |
|---------|------|------|---------|---------|----------------------|
| 获取媒体库视图 | GET | /Users/{UserId}/Views | UserId | JSON | 返回媒体库分类列表 |
| 获取媒体条目 | GET | /Users/{UserId}/Items | UserId, ParentId, IncludeItemTypes, SortBy, SortOrder, Filters, SearchTerm, Fields, limit, StartIndex | JSON | ParentId 为文件夹 ID |
| 获取影片详情 | GET | /Items/{ItemId} | ItemId, Fields | JSON | Fields 可指定返回字段 |
| 获取海报图片 | GET | /Items/{ItemId}/Images/{Type} | ItemId, Type (Primary/Backdrop/Logo), MaxWidth, MaxHeight, Tag | 图片流 | 直接作为 Image 组件源 |
| 搜索接口 | GET | /Search/Hints | SearchTerm, Limit | JSON | 支持关键字联想 |

### 3.3 播放相关

| 接口名称 | 方法 | 路径 | 主要参数 | 返回格式 | 鸿蒙 http 模块注意事项 |
|---------|------|------|---------|---------|----------------------|
| 获取视频流地址 | GET | /Videos/{ItemId}/stream | ItemId, MediaSourceId, DeviceId, PlaySessionId, static (true/false) | 重定向/M3U8/MPD | static=true 返回直链，false 返回 HLS/DASH |
| 获取字幕列表 | GET | /Videos/{ItemId}/{MediaSourceId}/Subtitles | ItemId, MediaSourceId | JSON | 返回可用字幕流信息 |
| 获取字幕文件 | GET | /Videos/{ItemId}/{MediaSourceId}/Subtitles/{StreamIndex}/Subtitle | StreamIndex, Format (srt/vtt/ssa/ass) | 文本流 | 下载后需解析渲染 |

### 3.4 进度同步相关

| 接口名称 | 方法 | 路径 | 主要参数 | 返回格式 | 鸿蒙 http 模块注意事项 |
|---------|------|------|---------|---------|----------------------|
| 同步播放进度 | POST | /Sessions/{SessionId}/Playing/Progress | SessionId, PositionTicks (100ns), IsPaused, Event | 空 | 需生成 PlaySessionId |
| 报告播放开始 | POST | /Sessions/{SessionId}/Playing | SessionId, ItemId, MediaSourceId | 空 | 开始播放时调用 |
| 报告播放停止 | POST | /Sessions/{SessionId}/Playing/Stopped | SessionId, ItemId, PositionTicks | 空 | 退出播放时调用 |

### 3.5 认证注意事项

**Emby 认证方式（两者选一）**：

**方式一：Header 认证**
```
X-Emby-Authorization: MediaBrowser UserId="xxx", Device="HarmonyOS", DeviceId="xxx", Version="1.0", Token="xxx"
```

**方式二：Query 参数认证**
```
GET /System/Info?api_key=your_api_key
```

**登录认证 Header 示例**：
```
X-Emby-Authorization: MediaBrowser Client="emby-harmonyos", Device="HarmonyOS", DeviceId="设备UUID", Version="1.0.0"
Content-Type: application/json
```

**登录成功返回**：
```json
{
  "User": { "Id": "xxx", "Name": "xxx" },
  "AccessToken": "xxx",
  "ServerId": "xxx"
}
```

**重要提示**：
- AccessToken 需存储用于后续请求
- 建议使用加密存储（AbilityStore）
- PlaySessionId 需每次播放时生成 UUID
- PositionTicks 单位为 100 纳秒（10,000,000 ticks = 1 秒）

## 4. 播放器核心能力要求

### 4.1 视频解码

| 能力项 | 要求 |
|-------|------|
| 播放器组件 | AVPlayer（首选）、VideoPlayer |
| 硬解支持 | 必须使用硬件解码 |
| 支持格式 | MP4、MKV、TS、MOV、WEBM |
| 编码支持 | H.264（必须）、H.265（必须）、VP9（推荐）、AV1（推荐） |
| 最大分辨率 | 4K@60fps（建议）、1080P@60fps（最低） |

### 4.2 HDR 支持

| HDR 类型 | 支持要求 | 实现方式 |
|---------|---------|---------|
| 杜比视界 Profile 5 | 推荐支持 | Media Kit / AVPlayer extended |
| 杜比视界 Profile 8 | 推荐支持 | Media Kit |
| HDR10 | 必须支持 | AVPlayer 内置 |
| HDR10+ | 推荐支持 | Media Kit |
| HLG | 推荐支持 | AVPlayer 内置 |

**注意事项**：
- HDR 需设备硬件支持
- 需检测屏幕 HDR 能力后决定是否启用
- 杜比视界需要特殊 License

### 4.3 音频直通

| 音频格式 | 支持要求 | 实现方式 |
|---------|---------|---------|
| Dolby Digital (AC3) | 必须支持 | AudioRenderer Passthrough |
| DTS | 必须支持 | AudioRenderer Passthrough |
| DTS-HD | 推荐支持 | AudioRenderer Passthrough |
| Dolby TrueHD | 推荐支持 | AudioRenderer Passthrough |
| 杜比全景声 | 推荐支持 | AudioRenderer Passthrough |

**注意事项**：
- 音频直通需 HDMI/eARC 或 USB-C 音频输出支持
- 需处理无直通能力时的降级播放（LPCM）
- 使用 AudioCapturer/AudioRenderer 实现

### 4.4 字幕渲染

| 字幕格式 | 支持要求 | 实现方式 |
|---------|---------|---------|
| SRT | 必须支持 | 内置解析 |
| VTT | 必须支持 | 内置解析 |
| ASS/SSA | 推荐支持 | 第三方库或自研 |
| 内嵌字幕 | 必须支持 | 需提取并渲染 |

**字幕样式调节**：
- 字体大小：50%-200%
- 字体颜色：白/黄/红/绿/自定义
- 背景颜色：透明/半透明/实色
- 字幕位置：底部/顶部
- 边缘描边：支持

### 4.5 播放控制

| 功能 | 规格 |
|-----|------|
| 倍速播放 | 0.5x、0.75x、1.0x、1.25x、1.5x、1.75x、2.0x、2.5x、3.0x |
| 画幅切换 | 16:9、4:3、原始比例、全屏拉伸 |
| 音画同步 | ±500ms 调整范围 |
| 快进/快退 | 10秒/15秒/30秒可选 |
| 断点续播 | 记住上次位置 |

### 4.6 网络自适应

| 协议 | 支持要求 |
|-----|---------|
| HLS | 必须支持（.m3u8） |
| DASH | 推荐支持（.mpd） |
| 直接流 | 推荐支持（.mp4 直链） |

**自适应策略**：
- 根据网速自动切换码率
- 支持手动选择码率
- 缓冲策略：预缓冲 10-30 秒
